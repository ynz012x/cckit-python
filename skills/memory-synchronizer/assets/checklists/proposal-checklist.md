# 更新提案自检清单

在向用户展示提案前，逐项检查：

## 作用域正确性

- [ ] 每个 CLAUDE.md 仅涉及其管辖范围内的变更
- [ ] 子目录有独立 CLAUDE.md 时，父级 CLAUDE.md 的提案不包含该子树的变更
- [ ] 变更文件的归属路由正确（最长前缀匹配）

## 变更覆盖

- [ ] 所有检测到的结构性变更都已映射到修改动作
- [ ] 无遗漏的高相关变更（新目录、新依赖、配置变更等）
- [ ] 已忽略的变更确实不影响项目级知识

## 增量一致性

- [ ] 未受影响的 section 保持原样（内容和格式不变）
- [ ] 修改范围最小化，仅改动必要的行
- [ ] 不引入与变更无关的格式调整

## 行数合规

- [ ] 每个 CLAUDE.md 修改后总行数 <= 200 行
- [ ] 如有拆分，外部文件路径在同级 `docs/` 目录下
- [ ] 拆分文件名使用 UPPER_SNAKE_CASE
- [ ] CLAUDE.md 中保留 section 标题和引用链接

## 内容合规

- [ ] 新增内容属于项目级知识（构建命令、编码规范、架构决策、命名约定、工作流）
- [ ] 不包含个人偏好、临时笔记或业务逻辑说明
- [ ] 新增内容与实际变更一致（如新 skill 描述与 SKILL.md 的 description 匹配）

## 格式正确

- [ ] 表格列对齐、行完整
- [ ] 代码块有语言标记且围栏闭合
- [ ] 标题层次一致（`##` 为顶层 section）
- [ ] 引用链接路径正确且可访问
