# 行数预算管理

## 基本约束

每个 CLAUDE.md 不得超过 **200 行**。此限制的目的是减少 AI 助手的上下文消耗，保持信息密度。

## Section 保留优先级

当 CLAUDE.md 需要缩减时，按优先级从高到低决定保留顺序：

| 优先级 | Section 类型 | 理由 | 典型行数 |
|--------|-------------|------|----------|
| P0 | 项目概览 | AI 理解项目定位的基础 | 3-8 行 |
| P1 | 常用命令 | 日常开发操作必备 | 10-25 行 |
| P2 | 架构概述 | 指导代码导航和理解 | 10-30 行 |
| P3 | 开发规范 | 避免 AI 生成不符合约定的代码 | 5-20 行 |
| P4 | 依赖 | 通常很短，性价比高 | 3-10 行 |
| P5 | 测试 | 可拆分不影响日常编码 | 5-15 行 |
| P6 | 工作流/部署 | 操作频率较低 | 10-30 行 |
| P7 | 详细说明/注意事项 | 参考性内容，适合外置 | 10-50 行 |

## 精简策略（拆分前优先尝试）

在决定拆分之前，先尝试以下精简措施：

1. **删除过时内容**：已移除的功能、废弃的命令、不再使用的依赖
2. **合并重复信息**：多处描述同一工具/流程的段落合并
3. **压缩表格描述**：表格中的说明列缩减为关键词而非完整句子
4. **移除显而易见的内容**：如"运行 `npm install` 安装依赖"这类不言自明的说明

## 拆分决策算法

当精简后仍超过 200 行：

1. 计算每个 section 的**拆分收益** = 行数 / 优先级权重（P0=8, P1=7, ..., P7=1）
2. 选择拆分收益最高的 section（即低优先级 + 高行数）
3. 将该 section 内容提取到外部文件
4. 在 CLAUDE.md 中替换为标题 + 一句概述 + 引用链接（约 3 行）
5. 重新计算行数，如仍超标则重复步骤 2-4

## 外部文件存放约定

拆分出的内容放在 CLAUDE.md **同级目录下的 `docs/` 子目录**：

```
./CLAUDE.md              → ./docs/ARCHITECTURE.md
./CLAUDE.md              → ./docs/CODING_STANDARDS.md
./backend/CLAUDE.md      → ./backend/docs/API_SPEC.md
```

- 文件名使用 **UPPER_SNAKE_CASE**，从 section 标题派生
- 如 `docs/` 目录不存在，创建之
- 如外部文件已存在（之前拆分过），更新其内容

## 引用链接格式

CLAUDE.md 中用标准 markdown 相对链接引用外部文件：

```markdown
## 架构

项目采用分层架构，详见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)。
```

保留 section 标题和一句话概述，确保 AI 助手无需读取外部文件即可知道该 section 的大致内容。

## 行数计算规则

- 所有行均计入行数（包括空行、标题行、代码块围栏）
- 代码块内容按实际行数计算
- 目标是严格控制在 200 行以内
