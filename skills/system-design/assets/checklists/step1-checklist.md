# Step 1 检查清单：明确需求

## 项目背景（Step 1.1）

- [ ] 项目目标已明确
- [ ] 启动原因已说明
- [ ] 成功标准已定义
- [ ] 项目范围边界已确定（做什么/不做什么）

## 主题域划分（Step 1.2 - SERU-S）

- [ ] 主题域数量合理（3-8个）
- [ ] 每个主题域有明确的名称和描述
- [ ] 每个主题域的核心职责已定义
- [ ] 主题域之间的关系已梳理
- [ ] 主题域关系图已绘制（Mermaid）
- [ ] 无职责重叠或遗漏

## 业务事件识别（Step 1.3 - SERU-E）

- [ ] 外部事件已识别（用户操作、外部系统调用）
- [ ] 时间事件已识别（定时任务、批处理）
- [ ] 状态事件已识别（状态变更触发）
- [ ] 每个事件有触发者、类型、频率
- [ ] 事件已按主题域归类
- [ ] 核心业务事件已标注

## 报表与管控点（Step 1.4 - SERU-R）

- [ ] 关键报表已识别
- [ ] 报表用途和使用者已明确
- [ ] 管控点（监控指标）已识别
- [ ] 报表更新频率已确定

## 用户确认

- [ ] 主题域划分已与用户确认
- [ ] 业务事件清单已与用户确认
- [ ] 报表和管控点已与用户确认
- [ ] 无遗漏的重要需求
