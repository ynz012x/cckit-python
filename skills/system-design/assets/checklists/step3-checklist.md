# Step 3 检查清单：勾勒架构

## 现有代码分析（Step 3.1）

- [ ] 代码库目录结构已扫描（如为增量设计）
- [ ] 技术栈已识别
- [ ] 分层架构已识别
- [ ] 现有模块划分已了解
- [ ] 代码规范已了解

## 架构对齐（Step 3.2）

- [ ] 新设计与现有架构风格一致
- [ ] 命名规范保持一致
- [ ] 目录组织保持一致
- [ ] 依赖注入方式保持一致

## 组件划分（Step 3.3）

- [ ] 组件基于主题域划分
- [ ] 每个组件职责单一、边界清晰
- [ ] 组件之间依赖关系明确
- [ ] C4 Container图已绘制（Mermaid）

## 通信机制（Step 3.4）

- [ ] 同步通信场景已确定（API/gRPC）
- [ ] 异步通信场景已确定（消息队列）
- [ ] 通信协议已选择
- [ ] 组件交互图已绘制（Mermaid）

## 架构决策（Step 3.5）

- [ ] 关键技术决策已记录（ADR）
- [ ] 每个决策有背景和理由
- [ ] 风险已识别和评估
- [ ] 替代方案已考虑

## 用户确认

- [ ] 组件划分已与用户确认
- [ ] 架构决策已与用户确认
- [ ] 无重大架构风险

## 歧义澄清

- [ ] 现有代码中设计意图不明确处已向用户确认
- [ ] 架构对齐方式的多种选择已与用户讨论确定
- [ ] 同步/异步通信方案的取舍已与用户确认
- [ ] 涉及重大取舍的架构决策已由用户拍板
