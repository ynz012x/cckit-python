# Step 5 检查清单：设计组件与接口

## 用例建模（Step 5.1 - SERU-U）

- [ ] 从业务事件转化为用例
- [ ] 参与者已识别（用户、系统、外部系统）
- [ ] 用例清单已完成
- [ ] 用例图已绘制（Mermaid）
- [ ] 用例按功能类型分类（业务/管理/维护）

## 用例细化（Step 5.2）

- [ ] 根据设计深度选择了细化范围
- [ ] 主成功场景已描述
- [ ] 扩展场景（异常流程）已考虑
- [ ] 前置条件和后置条件已定义
- [ ] 边界情况已考虑

## 服务接口设计（Step 5.3）

- [ ] 每个服务有明确的职责描述
- [ ] 方法签名完整（方法名、参数、返回值）
- [ ] 异常类型已定义
- [ ] 幂等性已标注
- [ ] 接口命名符合规范
- [ ] 接口设计表已完成

## 实体类设计（Step 5.4）

- [ ] 实体属性完整
- [ ] 业务方法已设计
- [ ] 构造函数/工厂方法已考虑

## DTO设计（Step 5.5）

- [ ] 请求对象已设计
- [ ] 响应对象已设计
- [ ] 字段校验规则已定义
- [ ] DTO与实体的转换关系已明确

## 核心流程设计（Step 5.6）

- [ ] 核心用例有序列图
- [ ] 序列图展示了组件间调用关系
- [ ] 关键数据流已标注
- [ ] 状态转换已设计（如有）
- [ ] 异常处理流程已设计

## 用户确认

- [ ] 用例覆盖度已与用户确认
- [ ] 接口设计已与用户确认
- [ ] 流程设计已与用户确认
- [ ] 无遗漏的重要接口

## 歧义澄清

- [ ] 用例的前置/后置条件模糊处已向用户确认
- [ ] 接口参数设计有多种方案处已与用户讨论确定
- [ ] 异常处理策略不明确处已与用户确认
- [ ] 核心流程步骤顺序或分支逻辑有歧义处已澄清
