# Step 4 检查清单：定义数据模型

## 实体识别（Step 4.1）

- [ ] 从业务事件和报表中提取了候选实体
- [ ] 核心实体数量合理（10-30个）
- [ ] 实体命名清晰、符合业务语义
- [ ] 无重复或冗余实体

## 实体映射（Step 4.2）

- [ ] 与现有代码的实体进行了对比（如为增量设计）
- [ ] 已存在的实体已标注
- [ ] 需新增的实体已标注
- [ ] 需修改的实体已标注
- [ ] 映射表已完成

## 属性定义（Step 4.3）

- [ ] 每个实体定义了核心属性（3-5个）
- [ ] 属性类型已确定
- [ ] 必填/可选已标注
- [ ] 属性说明清晰

## 关系建模（Step 4.4）

- [ ] 实体间关系已识别（1:1, 1:N, M:N）
- [ ] 关系命名有业务含义
- [ ] ER图已绘制（Mermaid erDiagram）
- [ ] 无循环依赖或不合理的关系

## 数据所有权（Step 4.5）

- [ ] 每个实体有唯一的写入者（服务）
- [ ] 读取者已列出
- [ ] 所有权矩阵已完成
- [ ] 跨服务数据访问方式已明确

## 用户确认

- [ ] 实体清单已与用户确认
- [ ] 实体映射已与用户确认
- [ ] 关系模型已与用户确认
- [ ] 无遗漏的重要数据

## 歧义澄清

- [ ] 语义不明确的实体名称已向用户确认含义
- [ ] 关系类型不确定处已与用户讨论确定
- [ ] 现有实体修改范围（新增字段/重构）已与用户确认
- [ ] 数据所有权归属有争议处已与用户确认
