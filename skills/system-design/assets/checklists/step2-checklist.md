# Step 2 检查清单：定义规模

## 性能需求（Step 2.1）

- [ ] 响应时间要求已定义（如 p99 < 200ms）
- [ ] 吞吐量要求已定义（如 1000 QPS）
- [ ] 并发量要求已定义（如 10000 同时在线）
- [ ] 性能指标可测量、可验证

## 数据规模（Step 2.2）

- [ ] 核心数据量已估算
- [ ] 数据增长率已估算
- [ ] 存储需求已估算
- [ ] 数据保留策略已考虑

## 可用性要求（Step 2.3）

- [ ] SLA目标已定义（如 99.9%）
- [ ] RTO（恢复时间目标）已定义
- [ ] RPO（数据恢复点目标）已定义
- [ ] 容灾要求已明确

## 技术约束（Step 2.4）

- [ ] 编程语言/框架已确定
- [ ] 基础设施要求已明确
- [ ] 现有系统集成要求已列出
- [ ] 技术栈与团队技能匹配

## 组织约束（Step 2.5）

- [ ] 团队规模已了解
- [ ] 团队技能栈已了解
- [ ] 时间约束已明确
- [ ] 预算约束已了解（如有）

## 用户确认

- [ ] 性能指标已与用户确认
- [ ] 技术约束已与用户确认
- [ ] 约束条件合理可行
